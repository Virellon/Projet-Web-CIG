# Projet Web 

## Site Pour la société Cubic Infrastructure Group

- ***En Construcion ...***


### ***Axe 1 : Le Portail Public (Vitrine & Communauté)***

#### Le site doit être la vitrine du serveur Minecraft.
- Système d'actualités : Affichage d'articles publiés par l'équipe (mises à jour, événements).
- Classements (Leaderboards) : Affichage de statistiques de jeu dynamiques (ex: Top joueurs, temps de jeu, économie). C'est à vous d'imaginer la structure de ces données.
- État de l'infrastructure : Un indicateur visuel (dynamique ou simulé) montrant si les serveurs de jeu et vocaux sont en ligne.

### ***Axe 2 : L'Espace Joueur (Extranet)***

#### Un espace sécurisé pour fidéliser la communauté.
- Authentification sécurisée : Inscription, connexion, et gestion des mots de passe (hashage obligatoire).
- Profil Utilisateur : Personnalisation du profil (avatar, description, réseaux sociaux liés).
- Historique : Visualisation des actions du joueur (historique d'achats dans la boutique, sanctions éventuelles reçues en jeu).

### ***Axe 3 : La Boutique Intégrée (E-commerce simplifié)***

#### Le nerf de la guerre pour la pérennité financière de Cubic.
- Catalogue dynamique : Affichage de produits (Grades VIP, cosmétiques, monnaie virtuelle) organisés par catégories.
- Panier virtuel : Utilisation avancée des sessions PHP pour gérer un panier d'achats (ajout, suppression, modification des quantités).
- Processus de commande : Simulation d'un tunnel de paiement (checkout) aboutissant à la génération d'une "facture" ou d'un récapitulatif en base de données.
### ***Axe 4 : Le Back-Office (Panel d'Administration)***

#### Un espace protégé réservé aux membres de l'équipe Cubic (gestion des rôles : Modérateur vs Administrateur).
- Gestion du contenu : Interface CRUD (Create, Read, Update, Delete) pour les actualités et les produits de la boutique.
- Modération : Interface de gestion des utilisateurs inscrits (modification des droits, bannissements web).
- Tableau de bord central : Des indicateurs de performance pertinents choisis par vos soins (chiffre d'affaires simulé, nombre d'inscriptions sur la semaine, etc.).

#### Exigéncés Téchniqués & Libérté d'Action
- Base de Données : Le schéma relationnel n'est pas fourni. C'est à votre équipe de concevoir les tables, les clés étrangères et les types de données les plus optimisés.
- Sécurité : Une tolérance zéro sera appliquée sur les failles basiques (Injections SQL, failles XSS, mots de passe en clair).
- Carte Blanche (Le Facteur X) : Pour remporter l'appel d'offres, votre équipe doit imaginer et intégrer une fonctionnalité majeure qui ne figure pas dans ce cahier des charges, mais qui apporterait une forte valeur ajoutée à Cubic Infrastructure Group. Si vous faites un choix technique différent de la demande, il sera nécessaire de fournir des explications.
